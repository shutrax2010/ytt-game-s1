# サンプルゲーム仕様：Tap Boxing（タップ・ボクシング）

3rd パーティ連携のサンプルゲーム。React 製・スコア制。プレイヤーは「ジャブ / ストレート / アッパー」をタップして相手を攻撃し、制限時間内に**何人倒せるか**を競う。連携プロトコルは `../3rd-game-dev-guide.md` / `../3rd-game-spec.md`（`pgp` postMessage）に準拠。

> このフォルダの `index.html` は連携確認用の最小プレースホルダ。本ゲーム（React ビルド）が最終的な成果物で、ビルド出力の `index.html` がこれを置き換える。

---

## 1. 概要

- **形式**: React（Vite）で作る standalone web アプリ。プラットフォームの iframe にロードされる。
- **ジャンル**: 一人称視点のタップ・ボクシング。相手を殴って倒す。
- **勝敗軸**: スコア制（`higher_is_better` / `points`）。**倒した相手の人数**が基本スコア。
- **プレイヤーのライフ**: なし。被弾しても倒れないが、一瞬**スタン**して手が止まる（＝手数が減り、スコアが伸びない）。
- **制限時間**: プラットフォーム管理（既定 180 秒）。時間切れで終了し、その時点の撃破数を報告。

---

## 2. ゲームルール

1. 目の前に相手が 1 人現れる。
2. 「ジャブ / ストレート / アッパー」をタップして攻撃し、相手のライフを削る。
3. 相手はガード・回避・攻撃をしてくるので、技を組み合わせて崩す。
4. 相手のライフを 0 にすると撃破。次の（少し強い）相手が現れる。
5. 制限時間まで繰り返し、**撃破数**を競う。プレイヤーが倒れることはない。

---

## 3. 攻撃技の特性

3 種の技は「速さ・威力・硬直・ガードへの通り」がトレードオフになっており、状況で使い分ける。

| 技 | 発生〜硬直 | ダメージ | 特性 |
| --- | --- | --- | --- |
| **ジャブ** | 速い（硬直短・連打可） | 小 | けん制の主軸。相手の攻撃予兆に**割り込んでキャンセル**できる。ガードにはほぼ通らない |
| **ストレート** | 中 | 中 | バランス型。崩れた相手への追撃に向く。ガード中は半減 |
| **アッパー** | 遅い（硬直長） | 大 | **ガードブレイク**（ガードを崩す）。当てれば大ダメージだが、外すと硬直が長く被弾しやすいハイリスク |

（数値は初期値・要調整。例：ジャブ 4 / ストレート 9 / アッパー 18）

---

## 4. 相手の行動と攻略

相手は状態を切り替えてくる。プレイヤーは相手の状態を見て技を選ぶ。

| 相手の状態 | 内容 | 対処 |
| --- | --- | --- |
| **通常** | 無防備 | どの技も通る。アッパーで大ダメージ |
| **ガード** | 防御姿勢。通常攻撃を軽減 | ジャブ・ストレートは効きにくい。**アッパーでガードブレイク**して崩す |
| **回避** | ウィービングで避ける（予兆あり） | 攻撃は空振りする。**攻撃を控えて**やり過ごす |
| **攻撃予兆 → 攻撃** | 溜めてから殴ってくる | 予兆中に**ジャブで割り込む**とキャンセル（カウンター）。間に合わなければ手を止めて被弾を避ける |

### 技 × 相手状態 ダメージ表（設計の核）

| 技 ＼ 相手 | 通常 | ガード中 | 回避中 | 攻撃予兆中 |
| --- | --- | --- | --- | --- |
| ジャブ | 小 | ほぼ無効 | 空振り | **割り込み成功**（相手の攻撃キャンセル＋小ダメージ） |
| ストレート | 中 | 半減 | 空振り | 遅く割り込めず被弾しやすい |
| アッパー | 大 | **ガードブレイク**（崩し＋小ダメージ＋よろけ） | 空振り | 最も被弾しやすい（硬直長） |

### 攻略の組み立て（例）

- ジャブで手数を稼ぎつつ様子見 → 相手が**ガード**したら**アッパーで崩す** → 崩れた隙に**ストレート/アッパー**で大ダメージ。
- **攻撃予兆**が出たら、ジャブで割り込んでカウンター、または手を止めて被弾（スタン）を回避。
- **回避**モーションが見えたら空振りを避けて技を温存する。

---

## 5. プレイヤーの被弾＝スタン

- プレイヤーにライフはない。相手の攻撃が当たると **約 0.5 秒スタン**（タップ入力を受け付けない・画面を軽く揺らす/赤フラッシュ）。
- スタン中は攻撃できないため手数が落ち、結果的にスコアが伸びない。これが唯一のペナルティ。
- 被弾は「相手の攻撃予兆を放置してタップし続けた」時に起きやすい設計にし、無闇な連打だけでは伸びず、緩急が要るようにする。

---

## 6. 相手の進行と難易度スケール

倒すごとに次の相手が登場し、少しずつ強くなる（時間切れまでエンドレス）。

| 段階が進むと | 変化 |
| --- | --- |
| ライフ | 増える（例：30 → 40 → 50 …） |
| 攻撃頻度 | 上がる（予兆が短く・間隔が詰まる） |
| ガード頻度 | 上がる（アッパーでの崩しがより重要に） |
| 回避頻度 | 上がる（空振りのリスク増） |

相手の見た目は色や体格で段階が分かるようにする（スプライトは差し替え可能）。

---

## 7. スコア

- 基本スコア = **撃破数**（`finalScore` に入れる）。表示は「◯人 撃破」。
- 同数の並びを避けたい場合の任意加点：連続ヒットのコンボボーナス、進行中の相手へ与えたダメージを端数として加算、など（要検討・§11）。
- プレイヤーは倒れないため、ゲームオーバーは無く、**時間切れ**（または GM 終了）でのみ終了する。

---

## 8. seed による公平性

- `pgp:init` で受け取る `seed` を乱数（mulberry32 等）に渡し、**相手の出現順・ガード/回避/攻撃の判断タイミングを固定**する。
- これにより全プレイヤーが**同じ相手を同じ挙動で**相手にでき、運ではなく操作の勝負になる。

---

## 9. pgp 連携（プラットフォーム対応）

| pgp | このゲームでの扱い |
| --- | --- |
| `ready` | React マウント完了時に送る（握手の起点） |
| `init` | `seed`（相手挙動の固定）と `durationSec`（残り時間表示）を受け取る |
| `start` | 受信でプレイ開始。それまで「準備中」表示 |
| `stop` | 時間切れ / GM 終了。入力を止め、集計して `gameover` を送る |
| `score` | 撃破するたびに現在の撃破数を送る（任意・A3 進行画面にライブ反映） |
| `gameover` | 終了時に `{ finalScore: 撃破数 }` を 1 回だけ送る（スコア制なので `cleared` は暗黙 true 扱い） |

登録用 JSON（管理画面からアップロード）例:

```json
{
  "id": "tap-boxing",
  "title": "Tap Boxing",
  "description": "ジャブ・ストレート・アッパーで殴って何人倒せるか競うタップ・ボクシング",
  "url": "https://tap-boxing.vercel.app/",
  "type": "score"
}
```

## 10. 画面構成（スマホ縦・一人称視点）

- **上部 HUD**: 撃破数（スコア）、残り時間、現在の相手の番号とライフバー。
- **中央 リング**: 相手を正面から表示（状態＝通常/ガード/回避/予兆/被弾/KO が見た目で分かる）。手前下にプレイヤーのグローブ。ヒット時のエフェクト（ヒットマーク・ダメージ数字）、「ガード!」「回避!」「カウンター!」のテキスト。
- **下部 操作**: 大きな 3 ボタン「ジャブ / ストレート / アッパー」（親指で連打しやすい配置）。スタン中はボタンを無効表示。
- 全画面 `100dvh`・`touch-action: none`・スクロール抑止（`../3rd-game-dev-guide.md` の守るべき項目に準拠）。

---

## 11. 技術構成と状態遷移

### React 構成（Vite）

- コンポーネント: `<Game>`（統括・pgp・タイマー）/ `<Hud>` / `<Ring>` + `<Opponent>` / `<Controls>`（3 ボタン）。
- 状態管理: `useReducer` でゲーム状態を集中管理。相手 AI・技の硬直・スタンは `requestAnimationFrame`（または `setInterval`）のティックで時間を進める。
- 乱数: `mulberry32(seed)` を 1 つ持ち、相手の判断・出現順に使う。
- pgp: `usePgp()` フックで `postMessage` を薄くラップ（`ready`/`onInit`/`onStart`/`onStop`/`score`/`gameover`）。
- アセット不要でも成立（CSS/絵文字/簡易 SVG）。後からスプライトに差し替え可能。

### 状態遷移

- **ゲーム**: `waiting`（start 待ち）→ `playing` → `ended`（時間切れ / stop）。
- **プレイヤー**: `ready`（入力可）→ `attacking`（技の硬直中・入力無視）→ `stunned`（被弾・約0.5秒入力無視）→ `ready`。
- **相手**: `idle` → （`guard` / `dodge` / `telegraph`→`attack`）→ `hit` → …、ライフ 0 で `ko` → 次の相手へ。

## 12. 未確定事項（要決定）

- 各技のダメージ・硬直、相手ライフ・難易度スケールの具体数値（プレイして調整）。
- スコアを「撃破数のみ」にするか、コンボ/与ダメージのボーナスを足して同点を割るか。
- スタン時間と、被弾判定の条件（予兆放置のみか、任意タイミングの攻撃にも当たるか）。
- 相手の種類バリエーション（見た目・くせのある挙動）をどこまで作り込むか。
- 連打の上限（ジャブ硬直をどこまで短くして「連打ゲー」に寄せるか）。
